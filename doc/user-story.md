# 1 问题描述
<!-- 工作目标、用户群体、解决问题 200字以内-->
**工作目标：** 我们希望做一个与数据获取、标注相关的众包平台。其中数据标注可能会对参与标注的人有一定的技能要求，比如标记牙片就需要牙科影像之类的知识；也可能数据标注对参与这会者并无特殊要求，如常见动物图片分类等等。

**用户群体：** 包括以下两类：
1. 希望能够利用闲散时间、不出家门且有趣味地赚外快的人（以下简称“参与者”）；
2. 希望有人能帮忙完成大量的重复的却无法由机器完成的任务的人（以下简称“发布者”）。

**解决问题：** 打破上述两类人的交流的阻隔，提供更好地平台方便两者。

# 2 需求获取
<!-- 要描述来源、途径、过程 -->

* 希望寻找懂技术的人帮忙实现自己需求的人（以下简称为“发布者”）。

### 3.1.1 编写个人简历
**必要性：** 高
**难度：** 适中
#### 故事卡片
作为程序员，我希望我能够编写我的个人简历方便他人认识我

#### 细节
- 能够设定自己的
  - [x] 用户名（作为该用户的标识，保证唯一性）
  - [ ] 昵称
  - [ ] 头像
  - [ ] 性别
  - [ ] 年龄
  - [ ] 学历
  - [ ] 工作经历
  - [ ] 擅长的技术（标签）
  - [ ] github账号
  - [ ] 个人博客链接
  - [ ] 主要项目经历
- 在本网站参与过的项目由服务器负责，本人无法进行更改

#### 测试
测试是否能正常编辑自己的个人简历


### 3.1.2 按技术查找项目
**必要性：** 高
**难度：** 适中
#### 故事卡片
作为程序员，我希望能按照我熟悉的技术查找项目以便于我筛选项目

#### 细节
- 首页有“项目类型”按钮，点击后展开一个列表，里面存放项目类型按钮，点击后返回对应项目列表

#### 测试用例
- 点击“项目类型”按钮，得到类型列表
- 点击其中一种类型，得到该类型的项目列表


### 3.1.3 按内容查找项目
**必要性：** 高
**难度：** 中上
#### 故事卡片
作为程序员，我希望能按照我熟悉或者感兴趣的内容查找以便于我筛选项目

#### 细节
- 网页右上角有搜索框，填入搜索内容后回车或者点击搜索按钮进行搜索
- 搜索内容长度不超过100字，否则自动截断并输出错误提示

#### 测试用例
- 正常输入，例如“微信墙”，“抢票”，得到正确类型列表
- 超过100字的输入，自动截断，得到错误提示

作为程序员，我能对一个的项目列表进行排序，例如按照经费、开发周期等

### 3.1.5 对项目提意见
**必要性：** 中上
**难度：** 高
#### 故事卡片
作为程序员，这么低的价格，这么短的天数要做这么大的项目，不可能的

#### 细节
- 项目内容页面有评论功能，可以对认为不合理的项目进行提出修改意见

#### 测试

### 3.1.6 申请加入一个项目
**必要性：** 高
**难度：** 适中
#### 故事卡片
作为程序员，我找到适合的项目后，我点击申请加入加入该项目

#### 细节
- 在每个任务详情页面都有一个“申请加入”按钮，点击后发送加入申请
- 每个人只能发送一次申请
- 申请加入后按钮变为“取消申请”，点击取消申请
- 不能取消未申请过或者已经加入的项目的申请

#### 测试用例
- 点击“申请加入”，成功申请
- 点击“取消申请”，取消申请
- 再次”申请加入”，成功申请


### 3.1.7 管理项目
**必要性：** 高
**难度：** 中上
#### 故事卡片
作为程序员，我想要取消申请或者我因某种原因不能再继续进行该项目要中途退出

#### 细节
- 在个人中心可以点击“管理项目”按钮查看当前所有我已经申请或者加入的项目
- 申请还未通过的项目右侧有“取消申请”按钮，点击后取消申请
- 已经加入的项目右侧有“退出”按钮，点击之后弹出确认对话框，点击确认发出退出申请，然后按钮变为“取消退出”

#### 测试用例
- 点击“取消申请”，取消申请
- 点击“退出”，确认对话框点击“确认”，发出退出申请
- 点击“取消退出”，取消退出请求
